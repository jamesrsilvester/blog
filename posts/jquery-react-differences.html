Differences Between jQuery and React

Let’s briefly touch on the similarities first. Both React and jQuery are javascript libraries that handle interface changes. The way they go about it is fundamentally different.

jQuery event handlers and corresponding UI changes are made by directly manipulating the existing HTML that has already been rendered to the DOM.

In contrast, event handlers in React do not directly affect the existing HTML. Instead those changes are recorded within the State object, which controls an intermediary layer commonly known as the “virtual” or “shadow” DOM. The HTML of the “Shadow” DOM and Actual DOM are then compared for differences. When a difference is found, the Actual DOM is partially re-rendered in order to reflect the requested update. Partial rendering is the key here, because it points to one of the main performance advantages of React. Because React instructs the page to re-render only the HTML for where a change is found, it removes the redundancy of having unchanged HTML load multiple times.
